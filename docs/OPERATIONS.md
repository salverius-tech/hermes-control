# Operations Runbook

## Local backend

```bash
python -m venv .venv
source .venv/Scripts/activate
python -m pip install -r requirements.txt
CONTROL_API_TOKEN=dev-token CONTROL_API_DB_PATH=./data/control-api.db uvicorn services.control_api.main:app --host 0.0.0.0 --port 8787
```

Use a real secret outside source control for anything beyond local development.

## Mobile development

```bash
cd apps/mobile
npm install
npm run typecheck
npm start
```

For physical Android devices, configure the mobile app with the host LAN IP/Tailscale hostname, not `127.0.0.1`.

## Verification commands

Fast backend-only check:

```bash
source .venv/Scripts/activate
python -m pytest services/control_api/tests -v
```

Full non-device canonical check:

```bash
source .venv/Scripts/activate
python scripts/verify.py
```

Android release build:

```bash
python scripts/verify.py --android
```

Connected-device release install/launch check:

```bash
python scripts/verify.py --android --sideload
```

If sideload fails with `no devices/emulators found`, unlock the phone, connect USB, enable USB debugging, and rerun `adb devices` until the device appears as `device`.


`apps/mobile/android/local.properties` is local-only and ignored by git.

## Release/debug APK notes

- Use release APKs for phone review because the JS bundle is embedded.
- Debug APKs may require Metro and can show splash/dev-client behavior if Metro is unavailable.
- Release builds need the Gradle memory settings in `apps/mobile/android/gradle.properties`.

## Troubleshooting

### API returns `401`

Check that the backend `CONTROL_API_TOKEN` matches the token saved in the mobile Settings screen.

### Mobile cannot connect from phone

Use the host machine LAN IP/Tailscale hostname and ensure the backend binds to `0.0.0.0`.

### Voice prompt button asks for permission or does nothing

Speech recognition uses `expo-speech-recognition`; native configuration is generated by Expo prebuild. Re-run native build verification after dependency/plugin changes.

### Android release build fails with memory/dexing errors

Stop Gradle and retry with the committed JVM settings:

```bash
cd apps/mobile/android
./gradlew --stop
./gradlew assembleRelease
```

### Real Hermes execution is missing

This is expected. Implement it behind `services/control_api/hermes_client.py` after selecting the integration surface: Hermes API Server, gateway, CLI, or another explicit adapter.
